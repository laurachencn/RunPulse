Audio Cues — Design Summary

Complete design for Phase 2 Audio Cues feature. Review and approve.

Configuration

Both Watch + iOS

Watch (Quick Toggles)

  • Master Voice Cues toggle
  • Pace, HR, Distance, Calories toggles
  • Distance interval selector (0.5/1/2/5 km)
  • Time interval selector (Off/1/5/10/15 min)

iOS (Full Configuration)

  • Same toggles with descriptions
  • Announcement type details
  • Frequency settings (distance + time)
  • Voice selection (Siri voice)

Metrics Scope: Standard

4 announcement types

PACE
"Your pace is 5 minutes 30 seconds per kilometer"
HEART RATE
"Your heart rate is 155 beats per minute"
DISTANCE
"You've run 3 kilometers"
CALORIES
"You've burned 180 calories"

Trigger Model: Both

Distance AND time intervals, independently configurable

DISTANCE INTERVAL
Default: Every 1 km
Options: 0.5, 1, 2, 5 km
TIME INTERVAL
Default: Off
Options: Off, 1, 5, 10, 15 min
Note: HR threshold alerts (from Phase 1) always fire independently — not affected by interval settings.

Architecture

Builds on existing VoiceService

New Components:
• AudioCueManager (Watch) — schedules and fires cues based on config
• AudioCueConfig (Shared model) — Codable settings synced via WCSession
• AudioCueSettingsView (Watch) — quick toggle UI
• AudioCuesSection (iOS) — added to SettingsView
Existing Components Used:
• VoiceService.speak() — for all announcements
• WatchConnectivityManager — sync config iOS ↔ Watch
• WorkoutManager — provides metrics for cue generation
